# PRODUCT_ONE_PAGER.md — TinyBookmarker

## Product sentence

TinyBookmarker is a tiny self-hosted bookmark manager for checking, cleaning and managing bookmarks.

## Who it is for

TinyBookmarker is for people who:

- save lots of bookmarks
- want control over favorites and organization
- need better cleanup than a browser bookmark bar offers
- want to review duplicates safely
- want to import and export without losing trust
- want a practical self-hosted setup

## What problem it solves

Most bookmark tools force a trade-off:

- either simple but too limited
- or feature-rich but too heavy, too unclear or too far from everyday bookmark work

TinyBookmarker should solve that by combining:

- fast save flow
- calm workspace
- strong organization
- safe duplicate cleanup
- trustworthy migration workflows
- lightweight self-hosting

## Product identity

TinyBookmarker is:

- bookmark-first
- favorites-aware
- organization-focused
- duplicate-safe
- migration-friendly
- self-host practical

TinyBookmarker is not:

- read-later-first
- archive-first
- reader-first
- note-taking-first
- knowledge-base-first
- dashboard-first

## Main concepts

### Bookmark
The main object.

### Favorite
Marks personal importance and quick access.

### Collection
Provides structural grouping.

### Tag
Adds flexible labeling across structure.

### Duplicate
Represents maintenance and cleanup work.

### Import / Export
Represents migration and browser roundtrip work.

### Profile / Settings / System / Admin
Secondary areas that support the product but do not define its identity.

## Main screens for the early product

1. **All Bookmarks**
2. **Favorites**
3. **Collections**
4. **Tags**
5. **Duplicates**
6. **Import / Export**

Secondary areas:
- Profile
- Settings
- System
- Admin

Primary action:
- **+ New Bookmark**

## Core daily workflow

1. Save a new bookmark
2. The bookmark stays in All Bookmarks as unsorted until it is organized
3. Organize it with favorites, collections and tags
4. Search or filter later
5. Review duplicates when needed
6. Import/export or roundtrip safely when migrating

## Version 1 scope

### Must have
- quick add
- bookmark list / grid
- favorites
- collections
- tags
- duplicate preflight
- duplicate review and safe merge
- import/export basics
- browser roundtrip basics
- lightweight self-host install/update flow

### Should not dominate v1
- archive workflows
- reader mode
- AI-heavy helpers
- team-first features
- enterprise controls

## UI direction

The UI should feel:

- calm
- focused
- understandable
- practical
- fast

It should avoid:

- tab overload
- too many equal-weight controls
- technical wording
- noisy dashboards
- feature-first clutter

## Success test

TinyBookmarker is on the right track if a user can quickly answer:

- Where are all my bookmarks?
- Where are my favorites?
- How do I organize them?
- How do I clean up duplicates?
- How do I import/export safely?
- What is the one main way to add a new bookmark?

If those answers are not obvious, the product is not clear enough yet.
